# Agent Skills - AI 智能助手技能库

欢迎使用 Agent Skills！这是一个专业的 AI 助手技能库,提供多个系统化、标准化的工作流技能,帮助你高效完成各类开发和创作任务。

## 📋 项目简介

Agent Skills 是一个精心设计的技能模块集合,每个技能都包含完整的工作流程、详细的阶段指南和参考文档。这些技能可以帮助你:

- 🚀 **系统化开发** - 从需求到代码的完整开发流程
- 📝 **专业文档创作** - PRD、技术文档、博客文章等
- 🎨 **界面原型设计** - 快速生成可交互的HTML原型
- 📊 **业务逻辑梳理** - 帮助产品经理理解和分析业务
- ⚡ **快速开发** - 适合小型任务的敏捷工作流

## 🎯 核心技能列表

### 1. 开发类技能

#### 🔧 Kiro - 完整开发工作流
**命令**: 参考 `commands/kiro.md`  
**技能**: `skills/kiro-skill/`

**适用场景**: 复杂功能开发、需要严格质量控制的项目

**六阶段工作流**:
1. 需求明确 - 将模糊想法转化为清晰的需求文档
2. 架构设计 - 基于需求创建技术实现方案
3. 任务规划 - 将设计分解为具体的编码任务
4. 代码实现 - 按照任务清单逐步实现功能
5. 代码复核 - 严格复核代码质量、规范一致性和需求符合度
6. 代码优化 - 基于复核报告进行迭代改进,确保代码质量

**特点**:
- ✅ 完整的需求到代码全流程管理
- ✅ 严格的代码复核和质量评分机制
- ✅ 支持多轮优化直到达标
- ✅ 完整的追溯链和文档记录

#### 🔥 Kiro Laravel - Laravel 专用开发工作流
**命令**: 参考 `commands/kiro-laravel.md`  
**技能**: `skills/kiro-laravel-skill/`

**适用场景**: Laravel 项目开发,与 Kiro 类似但针对 Laravel 优化

**特点**:
- ✅ 针对 Laravel 框架的特定规范和最佳实践
- ✅ 包含 Laravel 特有的代码示例和参考
- ✅ 完整的六阶段工作流

#### ⚡ Vibe - 快速开发工作流
**命令**: 参考 `commands/vibe.md`  
**技能**: `skills/vibe-skill/`

**适用场景**: Bug修复、小功能迭代、快速原型验证、代码重构

**三阶段工作流**:
1. 理解阶段 - 深入理解项目现状和用户需求
2. 规划阶段 - 与用户沟通方案,建立详尽任务列表
3. 执行阶段 - 按计划快速、高质量地完成任务

**特点**:
- ✅ 快速响应,灵活开发
- ✅ 保持高质量的代码规范
- ✅ 充分的上下文理解
- ✅ 适合敏捷开发场景

### 2. 产品类技能

#### 📋 PM - 产品经理业务逻辑整理助手
**命令**: 参考 `commands/pm.md`  
**技能**: `skills/pm-skill/`

**适用场景**: 整理现有业务逻辑、分析数据实体关系、评估需求变更影响

**三阶段工作流**:
1. 收集阶段 - 全面收集业务相关的数据实体信息、代码实现和表结构
2. 分析阶段 - 深入分析数据实体关系、业务规则和状态流转
3. 文档阶段 - 输出产品经理可直接使用的业务文档

**特点**:
- ✅ 使用业务语言而非技术语言
- ✅ 注重数据实体的维度分析
- ✅ 业务逻辑闭环原则(追溯上游、梳理下游、关联横向、覆盖异常)
- ✅ 先结论后详细的文档结构
- ✅ 支持快速模式和标准模式

#### 📄 PRD - B端供应链PRD文档生成器
**命令**: 参考 `commands/prd.md`  
**技能**: `skills/prd-skill/`

**适用场景**: 生成供应链PRD文档、产品需求梳理、功能模块规划

**六阶段工作流**:
1. 需求调研 - 深度挖掘用户需求、竞品分析和商业目标
2. 产品规划 - 创建产品架构、功能模块和优先级规划
3. 文档撰写 - 按照标准PRD结构展开详细需求文档
4. 需求评审 - 多维度评估PRD质量,重点关注可执行性
5. 文档优化 - 基于评审结果进行内容完善和优化
6. 发布准备 - 优化格式和制作最终可交付的PRD文档

**特点**:
- ✅ B端供应链场景专精
- ✅ 敏捷开发适配
- ✅ 支持现有系统功能增强
- ✅ 符合供应链行业最佳实践

## 🔧 工作原理

所有技能都遵循以下核心原则:

### 1. 阶段化工作流
每个技能都将复杂任务分解为多个阶段,每个阶段有明确的:
- 目标和产出
- 工作内容和步骤
- 完成标准和检查点

### 2. 用户主导
- 每个阶段完成后等待用户确认
- 支持在任何阶段返回修改
- 用户明确认可才进入下一阶段

### 3. 上下文保持
- 始终基于完整的前序文档工作
- 保持从开始到结束的完整追溯链
- 所有决策和产物都有明确的依据

### 4. 质量把关
- 每个阶段都有明确的质量标准
- 支持多维度的质量评估
- 提供具体的改进建议和优化方向

### 5. 规范遵循
- 严格遵守项目的代码规范
- 参考最佳实践和参考文档
- 保持一致的命名和结构

### Q: 如何选择合适的技能?
A: 参考上面的"技能选择指南"部分,根据任务类型、复杂度、时间要求等因素选择。

### Q: 可以中途切换技能吗?
A: 可以,但建议在阶段边界切换。例如从 Vibe 的理解阶段切换到 Kiro 的完整工作流。

### Q: 生成的文件保存在哪里?
A: 
- Kiro: `.claude/specs/{功能名}/`
- Vibe: 根据具体任务而定
- PM: `docs/业务文档/`
- PRD: `.claude/prd/{功能名}/`
- UI: `.claude/ui-generator/{功能名}/`
- Writer: `.claude/writers/{主题名}/`

### Q: 如何继续未完成的任务?
A: 说"继续上次任务",AI会检查对应目录,找到任务状态并继续。

### Q: 可以跳过某些阶段吗?
A: 不建议跳过阶段。如果确实需要快速完成,建议使用 Vibe 技能而不是跳过 Kiro 的阶段。
