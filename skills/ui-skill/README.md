# UI Skill - B端供应链静态交互HTML生成器

专业的B端供应链静态交互原型生成助手，帮助团队快速将产品需求转化为可交互的HTML原型。

## 快速开始

### 基本使用

直接在Claude Code中自然语言交互即可，无需记忆命令：

```
"我需要生成一个供应商列表页面"
```

或

```
"这是我们的PRD文档（附件），帮我生成静态交互原型"
```

Claude会自动调用UI Skill，通过六个系统化的阶段帮您生成高质量的HTML原型。

## 核心功能

### 1. 快速原型生成
将PRD文档或功能描述快速转化为可交互的HTML原型

### 2. 多模板支持
- **AT-UI风格**（默认）：现代、动感、丰富
- **Ant Design风格**：专业、简洁、高效
- **Element Plus风格**：清新、友好、易用

### 3. 完整交互模拟
- 完整的CRUD操作
- 表单验证
- 状态管理
- 模拟数据

### 4. B端供应链专业性
深度理解采购、库存、物流、供应商管理等核心业务场景

### 5. 开发友好
生成的代码规范清晰，可直接作为开发参考或基础

## 工作流程

UI Skill采用六阶段系统化工作流：

```
1. 需求理解 → 2. 界面规划 → 3. 模板选择
     ↓              ↓              ↓
4. 代码生成 → 5. 交互验证 → 6. 优化交付
```

### 阶段一：需求理解
- 识别输入类型（功能描述或PRD文档）
- 提取核心功能模块和页面类型
- 交互式确认需求理解
- 生成需求理解文档

### 阶段二：界面规划
- 设计页面布局和响应式方案
- 规划功能模块
- 设计交互流程
- 定义数据结构
- 生成界面规划文档

### 阶段三：模板选择
- 展示可用模板
- 推荐最合适的模板
- 用户确认选择
- 加载模板资源
- 生成模板选择记录

### 阶段四：代码生成
- 逐页面生成HTML代码
- 实现所有规划的功能
- 添加详细代码注释
- 确保代码可直接运行
- 生成使用说明文档

### 阶段五：交互验证
- 验证所有功能实现
- 测试交互功能
- 检查代码质量
- 测试浏览器兼容性
- 生成验证报告

### 阶段六：优化交付
- 根据验证反馈优化
- 完善文档
- 最终交付

```

```

## 常见问题

### Q: 如何开始使用UI Skill？

A: 直接在Claude Code中用自然语言描述您的需求即可：
- "生成一个采购订单列表页面"
- "这是PRD文档，帮我生成交互原型"

Claude会自动调用UI Skill并引导您完成整个流程。

### Q: 可以同时生成多个页面吗？

A: 可以。UI Skill会根据您的需求自动规划需要生成几个页面，并逐个生成。

### Q: 生成的HTML可以直接用于生产环境吗？

A: 不建议。UI Skill生成的是静态交互原型，主要用于：
- 需求验证和演示
- 团队沟通
- 开发参考

如需生产环境应用，需要进一步开发并对接真实后端API。

### Q: 可以修改生成的HTML吗？

A: 当然可以。生成的HTML代码结构清晰、注释完善，易于理解和修改。

### Q: 如何添加自定义模板？

A: 在 `templates/custom/` 目录下创建新文件夹，添加 `template.html` 和 `README.md` 即可。

### Q: 生成过程中可以中断吗？

A: 可以。UI Skill支持在任何阶段暂停，下次说"继续上次生成"即可继续。

### Q: 可以返回修改前面的阶段吗？

A: 可以。在任何阶段都可以要求返回修改前面的内容。
