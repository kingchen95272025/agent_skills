# 第六阶段：发布准备

## 阶段目标

优化格式，制作最终可交付的PRD文档，确保文档达到发布标准并便于团队使用。

## 前置条件

- PRD文档质量已达到8.5分以上，或用户明确满意当前版本
- 存在 `.claude/prd/{功能名}/draft.md` 或 `prd.md` 文件
- 所有主要问题已解决

## 工作内容

### 1. 优化文档结构和格式规范

#### 统一格式规范

- **标题层级**：确保标题层级正确，H1→H2→H3逻辑清晰
- **列表格式**：统一使用有序列表或无序列表
- **表格格式**：确保表格对齐，内容完整
- **代码块**：统一代码块的语言标识

#### 完善目录结构

```markdown
# 供应链系统PRD：{功能名} V1.0

## 目录

1. [版本迭代规划](#1-版本迭代规划)
2. [业务背景与目标](#2-业务背景与目标)
   2.1 [项目背景](#21-项目背景)
   2.2 [核心用户与场景](#22-核心用户与场景)
   2.3 [业务目标](#23-业务目标)
3. [业务名词](#3-业务名词)
4. [流程图](#4-流程图)
5. [功能需求详述与界面设计](#5-功能需求详述与界面设计)
6. [数据模型](#6-数据模型)
7. [验收标准](#7-验收标准)
```

#### 优化视觉呈现

- **使用分隔线**：在重要章节之间使用分隔线
- **突出重点信息**：使用粗体、引用块突出重点
- **统一图标**：使用统一的图标标识（📋、⚠️、✅等）
- **代码高亮**：确保代码块有正确的语法高亮

### 2. 完善图表显示效果

#### 检查Mermaid图表

确保所有Mermaid图表：
- 语法正确，可以正常渲染
- 布局清晰，易于理解
- 标签准确，无拼写错误
- 颜色搭配合理（如需要）

#### 检查图片占位符

确保所有图片占位符：
- 路径正确，格式统一
- 有清晰的说明文本
- 有插入提示信息

```markdown
![供应商管理页面](./images/supplier-management.png)
> 📋 **提示**：粘贴供应商管理页面原型图，包含筛选区、列表区和操作区
```

#### 优化表格展示

- 确保表格内容对齐
- 适当使用表格合并
- 表格不要过宽，必要时拆分
- 表格标题清晰明确

### 3. 完善版本信息

#### 添加文档元信息

```markdown
---
文档标题: 供应链系统PRD：{功能名}
文档版本: V1.0
创建日期: YYYY-MM-DD
最后更新: YYYY-MM-DD
文档状态: 已发布
负责人: [产品经理姓名]
评审人: [评审人姓名]
---
```

#### 添加变更历史

```markdown
## 变更历史

| 版本 | 日期 | 变更内容 | 变更人 |
|------|------|----------|--------|
| V1.0 | YYYY-MM-DD | 初始版本 | [姓名] |
| V1.1 | YYYY-MM-DD | [变更描述] | [姓名] |
```

### 4. 创建最终PRD文档

创建 `.claude/prd/{功能名}/final.md` 最终版本：

- 复制优化后的PRD内容
- 应用所有格式优化
- 完善版本信息
- 添加文档元信息
- 检查所有链接和引用

### 5. 创建交付目录和文件

#### 目录结构

```
.claude/prd/{功能名}/
├── final.md              # 最终PRD文档
├── research.md          # 需求调研报告（附件）
├── architecture.md      # 产品规划文档（附件）
├── review.md            # 评审报告（附件）
├── optimization_log.md  # 优化记录（附件）
├── README.md            # 文档阅读指南
└── images/              # 图片资源目录
    ├── .gitkeep        # 保持目录
    └── README.md       # 图片说明
```

#### 创建阅读指南

创建 `.claude/prd/{功能名}/README.md`：

```markdown
# {功能名}产品需求文档

## 文档概览

本目录包含{功能名}的完整产品需求文档及相关附件。

## 文档列表

### 核心文档

- **final.md** - 最终PRD文档（主文档）
  - 版本：V1.0
  - 状态：已发布
  - 用途：开发、设计、测试的主要参考文档

### 附件文档

- **research.md** - 需求调研报告
  - 用途：了解业务背景和用户需求

- **architecture.md** - 产品规划文档
  - 用途：了解产品架构和功能规划

- **review.md** - 需求评审报告
  - 用途：了解PRD质量评估和主要改进点

- **optimization_log.md** - 优化记录
  - 用途：了解文档优化历程

## 快速开始

### 产品经理

1. 阅读 `final.md` 了解完整需求
2. 参考 `architecture.md` 了解产品架构
3. 查看 `review.md` 了解质量评估

### 开发人员

1. 重点阅读 `final.md` 第5章"功能需求详述"
2. 参考 `final.md` 第6章"数据模型"
3. 参考 `final.md` 第7章"验收标准"

### 设计师

1. 重点阅读 `final.md` 第5章"界面设计"
2. 参考 `final.md` 第4章"流程图"
3. 参考 `architecture.md` 了解整体架构

### 测试人员

1. 重点阅读 `final.md` 第7章"验收标准"
2. 参考 `final.md` 第5章"业务规则"
3. 参考 `final.md` 第4章"流程图"

- **技术负责人**：[姓名] - [邮箱]
- **项目经理**：[姓名] - [邮箱]
```

#### 创建图片目录说明

创建 `.claude/prd/{功能名}/images/README.md`：

```markdown
# 图片资源目录

本目录用于存放PRD文档中引用的所有图片资源。

## 图片命名规范

- 使用kebab-case命名：`supplier-management-page.png`
- 包含模块前缀：`supplier-` `inquiry-` `price-`
- 说明图片类型：`-list` `-form` `-flow` `-model`

## 图片类型

### 页面原型图

- `supplier-management-list.png` - 供应商列表页
- `supplier-management-form.png` - 供应商表单页
- `inquiry-collaboration-list.png` - 询价列表页

### 流程图

- `supplier-admission-flow.png` - 供应商准入流程
- `inquiry-collaboration-flow.png` - 询价协同流程

### 数据模型图

- `supplier-data-model.png` - 供应商数据模型
- `inquiry-data-model.png` - 询价数据模型

## 图片格式要求

- **格式**：PNG或JPG
- **分辨率**：至少1920x1080
- **大小**：单个文件不超过5MB
- **清晰度**：确保文字清晰可读

## 添加新图片

1. 按照命名规范命名文件
2. 将文件放入本目录
3. 在PRD文档中引用
4. 更新本README文件
```

## 输出标准

### 文档完整性

最终交付包应包含：
- 最终PRD文档（final.md）
- 需求调研报告（research.md）
- 产品规划文档（architecture.md）
- 评审报告（review.md）
- 优化记录（optimization_log.md）
- 阅读指南（README.md）
- 图片资源目录（images/）

### 文档质量

- 内容完整准确
- 格式统一规范
- 结构清晰合理
- 易于阅读理解
- 便于团队使用

### 可用性

- 开发人员能理解要实现什么
- 设计师能设计界面原型
- 测试人员能编写测试用例
- 产品运营能理解业务逻辑

1. 向您展示最终PRD文档

- 发布前要全面检查文档质量
- 确保所有格式统一规范
- 完善版本信息和元数据
- 创建完整的交付目录
- 提供清晰的阅读指南

- 维护文档版本历史
- 定期review文档的准确性
- 归档旧版本文档

### 持续改进

- 收集使用过程中的问题
- 优化文档结构和内容
- 完善文档模板
- 提升PRD质量
