# Vibe Skill 快速开发工作流

这是一个快速开发的 Claude Code Skill，帮助您快速、高效地完成开发任务，同时保持代码质量。

## Skill 结构

```
vibe-skill/
├── SKILL.md                 # 主文件：核心工作流程和全局约束
├── README.md                # 本文件：使用说明
└── phases/                  # 阶段指南文件夹
    ├── 01-understand.md     # 第一阶段：理解
    ├── 02-plan.md           # 第二阶段：规划
    └── 03-execute.md        # 第三阶段：执行
```

## 何时使用这个 Skill

当你需要：

- 快速实现小功能
- 修复 Bug
- 代码重构
- 快速原型验证
- 小型迭代开发

Claude 会自动识别并使用这个 Skill。

## 三阶段工作流

```
理解 (Understand) → 规划 (Plan) → 执行 (Execute)
```

1. **理解** - 深入理解项目现状、代码规范和用户需求
2. **规划** - 与用户沟通方案，建立详尽的任务列表
3. **执行** - 按计划快速、高质量地完成任务

## 与六阶段工作流的对比

| 场景 | 推荐工作流 |
|------|-----------|
| 复杂新功能开发 | kiro-laravel-skill (六阶段) |
| 快速原型验证 | vibe-skill (三阶段) |
| 小功能迭代 | vibe-skill (三阶段) |
| Bug 修复 | vibe-skill (三阶段) |
| 代码重构 | vibe-skill (三阶段) |
| 需要严格质量控制 | kiro-laravel-skill (六阶段) |

## 全局约束

- 使用 ultrathink 模式深度思考
- 必须完全了解项目代码和规范
- 语言：简体中文
- 严格遵守 `.cursor/rules` 目录规则
- 执行任务前建立 To-dos，精确到每个子任务
- 不自动执行 SQL 语句
- 从全局到细节的方法论
- 按业务模块开发，不按代码分层

## 使用示例

### 开始新任务

```
用户：帮我实现一个商品库存预警功能
Claude：好的，让我先理解一下项目现状...
       [理解阶段]
       基于你的需求，我建议这样实现...
       [规划阶段，建立 todos]
       任务列表已经建好了，确认后我开始执行？
```

### 快速修复

```
用户：订单列表接口返回数据格式不对
Claude：让我先看看相关代码...
       [快速定位问题]
       找到问题了，我来修复它...
       [建立简单 todos，快速修复]
```

## 核心特点

1. **充分理解** - 开始前深入理解项目和需求
2. **提前沟通** - 规划阶段和用户充分沟通方案
3. **任务透明** - 详尽的 todos 让用户了解进度
4. **规范优先** - 严格遵守项目代码规范
5. **快速迭代** - 保持高效但不牺牲质量
6. **主动完成** - 主动推进工作，减少用户推动

## 阶段文件说明

### phases/ 目录

包含三个阶段的详细实施指南：

- **01-understand.md** - 详细说明如何理解项目和需求，包括规范收集、代码分析
- **02-plan.md** - 详细说明如何与用户沟通方案，如何建立详尽的任务列表
- **03-execute.md** - 详细说明如何高效执行任务，包括任务流程、代码规范、进度更新

## 注意事项

1. **理解阶段不能跳过** - 必须先理解项目规范
2. **规划阶段需要用户确认** - 方案和任务列表需要用户认可
3. **任务编号必须清晰** - 格式为"编号 + 空格 + 任务描述"
4. **实时更新进度** - 让用户始终了解当前状态
5. **SQL 脚本需要用户手动执行**

## 工作风格

Vibe 采用轻松、友好的交互方式：

- 用自然语言沟通，像和同事交流
- 提供解决方案导向的建议
- 主动完成工作，减少用户推动
- 保持积极乐观的态度

通过这种快速、高效的协作方式，确保在保持代码质量的同时大幅提升开发效率。
