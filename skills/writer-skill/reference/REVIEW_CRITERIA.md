# 质量评分标准

本文档详细说明Writer工作流中质量复核阶段的评分标准和机制。

## 评分机制概述

质量复核采用多维度评分机制，包括5个维度，每个维度0-10分，最终计算加权综合得分。

### 权重分配

- **内容质量**：25%权重
- **表达质量**：20%权重
- **结构质量**：20%权重
- **AI味评估**：20%权重
- **实用性评估**：15%权重

### 综合得分计算

```
综合得分 = 内容质量×0.25 + 表达质量×0.20 + 结构质量×0.20 + AI味评估×0.20 + 实用性评估×0.15
```

### 得分等级划分

- **≥ 8.5分**：优秀，可直接进入排版美化
- **6.0-8.4分**：良好，建议进行优化改进
- **< 6.0分**：需改进，必须进行优化

---

## 维度一：内容质量评估（25%）

### 评估要素

1. **信息准确性**（40%）
   - 事实、数据、引用是否准确
   - 信息是否来源可靠
   - 是否存在错误或误导性内容

2. **逻辑完整性**（30%）
   - 论证是否完整
   - 逻辑链条是否连贯
   - 是否有遗漏的关键环节

3. **价值密度**（30%）
   - 有效信息占比
   - 是否有冗余或无价值内容
   - 每个部分是否有明确的价值输出

### 评分标准

| 分数段 | 标准描述 |
|--------|----------|
| 9-10分 | 信息准确无误，逻辑完整严密，价值密度极高，每个部分都有显著价值输出 |
| 7-8分 | 内容准确完整，逻辑清晰，有较高价值，极少冗余 |
| 5-6分 | 基本准确，逻辑尚可，有一定价值，存在部分冗余 |
| 3-4分 | 部分准确，逻辑有明显缺陷，价值有限，冗余较多 |
| 0-2分 | 准确性差，逻辑混乱，价值低，充斥无用信息 |

### 评估清单

- [ ] 核对了所有事实和数据的准确性
- [ ] 验证了信息来源的可靠性
- [ ] 检查了论证的完整性
- [ ] 评估了逻辑链条的连贯性
- [ ] 分析了价值密度和有效信息占比
- [ ] 识别了冗余或无价值的内容

---

## 维度二：表达质量评估（20%）

### 评估要素

1. **语言流畅度**（40%）
   - 表达是否自然流畅
   - 句子是否通顺
   - 阅读是否顺畅

2. **可读性**（30%）
   - 是否易于理解
   - 段落长度是否适中
   - 句子结构是否简洁

3. **专业性**（30%）
   - 用词是否专业准确
   - 是否符合领域规范
   - 专业度是否适合目标读者

### 评分标准

| 分数段 | 标准描述 |
|--------|----------|
| 9-10分 | 语言流畅自然，可读性极佳，专业而易懂，表达精准有力 |
| 7-8分 | 表达流畅，可读性好，专业度适中，基本无阅读障碍 |
| 5-6分 | 基本流畅，可读性尚可，专业度一般，偶有阅读障碍 |
| 3-4分 | 表达生硬，可读性差，专业度不足，阅读体验不佳 |
| 0-2分 | 语言混乱，难以阅读，缺乏专业性，无法理解 |

### 评估清单

- [ ] 通读全文，评估整体流畅度
- [ ] 检查句子是否通顺自然
- [ ] 评估段落长度和结构的合理性
- [ ] 判断专业术语使用是否准确规范
- [ ] 确认专业度是否符合目标读者水平
- [ ] 识别生硬或难懂的表达

---

## 维度三：结构质量评估（20%）

### 评估要素

1. **层次清晰度**（40%）
   - 标题层级是否清晰
   - 结构是否一目了然
   - 内容组织是否有序

2. **逻辑连贯性**（30%）
   - 部分之间的逻辑关系是否明确
   - 过渡是否自然
   - 是否有逻辑跳跃

3. **重点突出度**（30%）
   - 核心内容是否突出
   - 重点信息是否明确
   - 主次是否分明

### 评分标准

| 分数段 | 标准描述 |
|--------|----------|
| 9-10分 | 层次极其清晰，逻辑连贯完美，重点突出显著，结构设计精妙 |
| 7-8分 | 结构清晰，逻辑连贯，重点明确，组织合理有序 |
| 5-6分 | 结构基本合理，逻辑尚可，重点不够突出，存在小的结构问题 |
| 3-4分 | 结构混乱，逻辑跳跃，重点不明，组织杂乱无章 |
| 0-2分 | 缺乏结构，逻辑不通，无法识别重点，完全混乱 |

### 评估清单

- [ ] 检查标题层级的清晰度和规范性
- [ ] 评估整体结构的合理性
- [ ] 验证部分之间的逻辑关系
- [ ] 检查过渡的自然性和流畅性
- [ ] 确认核心内容是否得到突出
- [ ] 判断主次关系是否分明

---

## 维度四：AI味评估（20%）

### 评估要素

1. **自然表达度**（40%）
   - 表达是否像人类写作
   - 是否有机械感
   - 语言是否生动自然

2. **人性化程度**（30%）
   - 是否有个人观点和见解
   - 是否有情感色彩
   - 是否有对话感

3. **套路化程度**（30%）
   - 是否使用AI常用套路表达
   - 是否有模板化痕迹
   - 是否千篇一律

### 评分标准

| 分数段 | 标准描述 |
|--------|----------|
| 9-10分 | 表达完全自然，充满人性化，无AI痕迹，独特性强，富有个性 |
| 7-8分 | 基本自然，有一定人性化，较少套路化表达，有部分独特见解 |
| 5-6分 | 部分自然，人性化不足，有一些AI味，存在套路化表达 |
| 3-4分 | 套路化明显，AI味较重，机械感强，缺乏个性 |
| 0-2分 | 充满AI套路，严重缺乏人性化，完全模板化，毫无个性 |

### 常见AI味特征识别

#### 套路化开头结尾
- "在当今这个...时代"
- "随着...的发展"
- "总而言之"
- "综上所述"

#### 机械化连接
- "首先...其次...再次...最后"
- "一方面...另一方面"
- "不仅...而且"（过度使用）

#### 空洞化表达
- "显而易见"
- "毋庸置疑"
- "不难发现"
- "可以看出"

#### 过度概括
- "众所周知"
- "大家都知道"
- "普遍认为"
- "一般来说"

#### 缺乏具体性
- 只有抽象概念
- 缺少具体细节
- 没有真实案例
- 缺乏个人视角

### 评估清单

- [ ] 识别套路化的开头和结尾
- [ ] 检查机械化的逻辑连接
- [ ] 发现空洞化的表达
- [ ] 识别过度概括的描述
- [ ] 评估内容的具体性和真实性
- [ ] 判断是否有个人观点和见解
- [ ] 确认是否有情感色彩和对话感

---

## 维度五：实用性评估（15%）

### 评估要素

1. **可操作性**（40%）
   - 方法是否具体可行
   - 步骤是否清晰明确
   - 读者能否照做

2. **实践指导价值**（35%）
   - 是否提供实际指导
   - 是否有真实案例
   - 是否有工具推荐

3. **落地可行性**（25%）
   - 方法是否现实可行
   - 是否考虑实际约束
   - 是否易于实施

### 评分标准

| 分数段 | 标准描述 |
|--------|----------|
| 9-10分 | 高度可操作，实践指导价值极高，完全可落地，提供详细步骤和工具 |
| 7-8分 | 可操作性好，有实践指导价值，基本可落地，有案例和方法 |
| 5-6分 | 有一定可操作性，部分可落地，指导价值一般 |
| 3-4分 | 可操作性差，难以落地，缺乏实践指导 |
| 0-2分 | 空洞抽象，无法落地，毫无实践价值 |

### 评估清单

- [ ] 检查方法的具体性和可行性
- [ ] 评估步骤的清晰度和完整性
- [ ] 判断读者能否按照指导实施
- [ ] 确认是否有真实案例支撑
- [ ] 检查是否提供了工具和资源
- [ ] 评估方法的现实可行性

## 评分注意事项

1. **客观公正**：基于标准评分，避免主观偏见
2. **具体详细**：每个维度都要有具体的分析和举例
3. **建设性**：不仅指出问题，还要提供改进建议
4. **可操作**：建议要具体可行，便于后续优化
5. **一致性**：前后评分标准保持一致
6. **全面性**：不遗漏任何重要的评估要素

## 常见评分误区

1. **过于宽松**：所有文章都评8分以上
2. **过于严格**：总是挑毛病，评分过低
3. **主观臆断**：基于个人喜好而非客观标准
4. **前后不一**：同类问题评分标准不统一
5. **缺乏依据**：给出分数却没有具体说明
6. **避重就轻**：只关注小问题，忽视大问题

## 评分最佳实践

1. **先通读全文**，形成整体印象
2. **按维度逐一评估**，不要遗漏
3. **记录具体问题**，标注位置
4. **提供改进建议**，具体可行
5. **计算综合得分**，给出结论
6. **与用户沟通**，确认理解一致
